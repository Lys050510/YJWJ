## [2026-07-22] — 模块八保底修复与体验优化

### 修复 — 总设置保底状态与统计面板（第二次更新）
- **取消总设置校验 clamp 逻辑**：移除 `totalDraws` 修改时自动 clamp 子计数器的限制，仅保留 NaN/负数→0 兜底，方便用户复刻游戏内真实开箱记录
- **总设置特殊保底改为下拉选择**：从简单 checkbox 改为"未触发 / 已触发"下拉，同时原子设置 `specialPityActive` 和 `specialPityUsed` 两个标志
- **总设置新增极品外观保底字段**：有 `extraGoldPity` 的宝箱（裂变/燃耀/天工/侠风/新春宝灯/新春华灯/永昼共 7 个）显示极品保底计数+已触发次数/14
- **总设置新增常驻红保底字段**：神骏宝炉显示常驻红保底计数+已触发次数/3
- **修复旧数据导致统计面板与设置弹窗状态不一致的 Bug**：旧版 checkbox 只修改 `specialPityActive`，新版下拉改用 `specialPityUsed`，统计面板同步切换为 `specialPityUsed` 判断，消除跨箱"假已触发"显示
- **修复神骏宝炉特殊保底已触发后红保底显示 100 而非 400**：新增常驻红保底优先级判断，`specialPityUsed` 后自动切换到 `permRedPity` 倒计时
- **修复双重保底宝箱特殊保底触发后仍显示红保底行**：锦鳞宝囊/化龙宝珠/玲珑宝莲/奇珍宝箱的一次性特殊保底触发后隐藏红保底行
- **修复无特殊保底/常驻红保底的宝箱仍显示「特殊保底状态 —」**：该行按需隐藏

### 优化 — 统计面板文案与布局
- 红色保底行文案改为「N次内必得神品外观（可重复触发）」，循环保底显示灰色"（可重复触发）"，特殊保底不显示
- 紫色保底行文案改为「N次内必得优品及以上外观（可重复触发）」
- 极品保底行文案改为「N次内必得极品外观（已触发 M/14）」
- 移除统计面板"红色保底""紫色保底""极品保底"标签前缀，保留自解释描述文案
- 特殊保底行保留"特殊保底"标签前缀，无紫色保底时隐藏紫色保底行

### 新增 — 宝箱列表拖拽排序
- 左侧宝箱列表支持拖拽排序（HTML5 Drag API），拖拽时半透明+目标金色光效
- 排序自动保存到 `localStorage.boxOrder`，刷新不丢失，新宝箱追加到末尾

### 修复
- **修复双重保底宝箱特殊红保底永不触发的严重 Bug**：化龙宝珠、锦鳞宝囊、玲珑宝莲、奇珍宝箱、神骏宝炉 5 个宝箱的「100 抽必出神品」机制完全失效，根因是 `redPityCounter` 仅在循环目标为红色时递增，而双重保底宝箱的循环目标是紫色
- **修复神骏宝炉常驻红色保底（400 抽，最多 3 次）完全失效**：连锁 Bug，因特殊保底永不触发导致前置条件 `specialPityUsed` 永远为 false
- **修复双重保底宝箱统计面板红保底显示为 "—"**：改为优先取循环红保底、其次取特殊红保底倒计时
- **修复无特殊保底宝箱的设置弹窗显示无意义的「特殊保底」复选框**
- **修复总设置按钮点击无反应**：`pity` 变量在使用前未声明导致 ReferenceError
- **修复保底触发 off-by-one**：所有 5 个保底条件从 `counter >= pityCount` 改为 `counter + 1 >= pityCount`，修复显示"1次内必得"时实际还需 2 次抽奖才触发保底的逻辑错误（计数器在抽奖之后才递增，导致触发条件落后显示 1 步）

### 优化
- 展示结果按品质从高到低排列（神品→极品→优品→良品），同品质保持抽取顺序
- 展示界面自动确认时间从 4/7 秒延长至 2 分 30 秒
- 紫色保底逻辑：抽出紫色或更高品质时重置计数（高品质可打断低保底）
- 开箱历史记录新增抽奖类型标签（单抽/十连抽/新春按钮名），统一金色高亮显示
- **单抽神品展示简化**：移除结果展示中的大号"神品"文字叠层（`#lootbox-big-text`）和结果卡片下方的保底标签（🔒循环保底等），神品展示仅保留品质标签+物品名，与其他品质一致

### 新春系列动画与 UI 优化
- **新春系列定格悬念动画**：减速阶段（六次方缓出）末尾随机触发"近在咫尺"效果——球列滑过目标上方球再拉回，模拟险些抽中的紧张感。触发概率基于上方球品质（红 95%、金 80%、紫 55%），滑过幅度 0.40~0.80 球径，使用 `sin²(πp)*(1-p)` 缓动函数确保起止平滑
- **新春系列按钮布局**：从 2×2 改为 1×4 横排（日月长明 / 山川秀润 / 风调雨顺 / 国泰民安），万事如意独立一行。四列抽奖区域对应四个按钮，单抽在某列播放，四抽在四列同时播放
- **抽奖区域视觉修复**：移除画布外圈聚光灯 `box-shadow`，画布 z-index 提升至最顶层，列背景不透明度提升至 0.88~0.95，消除 bg.png 暗角遮挡动画的问题
- **十连抽前 6 个光点加速**：间隔从 15ms 二次方递增至 150ms（与第 7 个光点速度对齐），不再匀速等待
- **总设置已抽次数即时同步**：修改总抽数后立即刷新相关保底计数器（不超过总抽数）、设置面板输入框、统计面板及宝箱列表
- **抽奖区域背景 WebP 化**：`assets/box/bg.png` → `bg.webp`（554KB → 62KB，节省 89%）
- **新春系列列间距缩小**：四列从均匀分布（间距 37px）→ 紧凑居中（间距 12px），列总宽 388px；1×4 按钮和万事如意按钮同步收窄至 `max-width: 388px` 对齐画布
- **心愿面板布局优化**：心愿图片移至标题行最右方，移除"未设置"文字提示，保留 `?` 占位符

## [2026-07-22] — 模块八音效系统

### 新增
- **Web Audio API 程序化音效**：不依赖任何外部音频文件，纯代码生成全部音效
- **音效开关按钮**：抽奖区域右上角 🔊/🔇 切换，状态通过 `localStorage` 持久化
- **四种音效类型**：
  - 宝箱摇晃声：白噪声 + 带通滤波 + LFO 振幅调制，模拟低沉隆隆声
  - 光球飞出音效：5 档品质差异化（蓝"噗"→红"哗"+升调序列），每个光球出现时触发
  - 揭示卡片音效：5 档品质（蓝单音→红辉煌和弦+上行音阶），结果展示时播放
  - 新春球滚动/定格：递归 tick 调度，随减速曲线自动降速；定格声低中高频三重叠

### 优化
- **预生成缓冲消除掉帧**：3 个共享噪声缓冲（100ms/20ms/8ms）首次初始化，动画期间零 `createBuffer` 调用，解决频繁 GC 导致的 rAF 丢帧
- 代码量从 650 行精简至 348 行（减少 46%），复用 3 个工具函数（`_playNoiseWhoosh`/`_playTone`/`_ensureBuffers`）

### 新增文件

## [2026-07-19] — 修复全新环境初始化 Bug

### 修复
- **修复全新环境首次访问全部功能失效的严重 Bug**
  - 根因：`config.js` 初始化时，空 localStorage 被设为 `{configVersion:0}`（truthy），导致三元表达式错误地取到 `undefined`，`activeHeroNames` 等 4 个字段变为 `undefined` 并持久化到 localStorage
  - 影响范围：新浏览器、无痕窗口、手机端首次访问 — 英雄/武器抽取按钮无反应、人员头像不显示、计分板英雄头像不显示
  - 桌面端已访问过的浏览器不受影响（localStorage 已有正确缓存走 else 分支）
  - `configVersion` 升级至 16，自动修复已损坏的缓存
  - 为迁移分支和 else 分支补充 9 处缺失字段的兜底检查
